# Milestone v0.3.0: Operational Excellence

**Status:** ✅ SHIPPED 2026-07-13

## Overview

Harden code quality, enrich observability, and add API documentation to make ecs-sd production-grade and self-describing. Target features: CacheSnapshot atomicity, zero panics in HTTP paths, rich health endpoints with K8s probes, 7 new Prometheus metrics, config endpoint with churn protection, OpenAPI/Swagger documentation, and comprehensive test coverage.

## Milestone Summary

**Key Decisions:**

- Decision: CacheSnapshot — Replace three separate `Arc<RwLock<...>>` with single `Arc<RwLock<CacheSnapshot>>` (Rationale: atomic replacement, prerequisite for health enrichment)
- Decision: Custom HTTP metrics middleware — `from_fn_with_state` Tower middleware instead of `axum-prometheus` (Rationale: incompatible registry ecosystem)
- Decision: `utoipa` for OpenAPI — 3 new crates, simpler `merge(SwaggerUi::new(...))` pattern (Rationale: minimal deps, good axum support)
- Decision: `/health/live` for ALB — Must point at always-200 endpoint, not `/health` (Rationale: prevents eviction loop during AWS outage)
- Decision: `last_manual_refresh_request` as separate `AtomicU64` — Must NOT fold into CacheSnapshot (Rationale: rate-limiting concern, different write path)
- Decision: `discover_all_clusters` returns `Result<Vec<Target>, DiscoveryError>` — caller skips cache replacement on `Err` (Rationale: stale-while-revalidate guarantee)

**Issues Resolved:**

- CacheSnapshot atomicity — torn reads eliminated
- Panics in HTTP response construction removed
- SDK version drift prevented via exact pins
- ALB eviction loop during AWS outages prevented
- Target cache wiped on transient AWS failures — fixed with churn protection

**Technical Debt Incurred:**

- PKG-03: GHCR auto-push / GitHub Actions release not fully wired
- WR-03: `publish_cache_to_gossip` holds snapshot lock across async gossip awaits — should clone then release
- AWS credential modes: E2E testing incomplete
