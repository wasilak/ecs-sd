<objective>
Add a Tower middleware that intercepts every matched HTTP request, records its latency and final status, and tags the metrics with the axum `MatchedPath` pattern (not the raw URI). Satisfies MET-08 and MET-09.

Purpose: Per-request observability requires access to `AppState` (for the metric handles) AND the response status, which only a `from_fn_with_state` middleware sees. Using the matched path pattern is the critical control that prevents UUID cardinality explosion on proxy routes.
Output: A new `src/middleware/` module with `http_metrics_middleware`, wired onto the merged router via `.route_layer()` so `MatchedPath` is populated before the middleware fires.
</objective>
