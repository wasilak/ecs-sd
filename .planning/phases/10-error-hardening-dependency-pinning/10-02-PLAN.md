<objective>
Make the AWS SDK dependency versions deterministic and make a missing AWS region a hard, visible startup failure instead of a silent us-east-1 default.

Purpose: `aws-sdk-ec2` and `aws-sdk-ecs` must move together as a matched release-series pair (both currently share `aws-smithy-runtime 1.11.3`); a loose `"1.0"` range lets `cargo update` silently pull a mismatched EC2 SDK. Separately, silently defaulting a mis-configured deployment to `us-east-1` can point the service at the wrong region/account and produce confusing empty discovery — it should fail loudly at boot.

Output: `Cargo.toml` exact-pins `aws-sdk-ec2 = "=1.236.0"` (and, for matched-series determinism, `aws-sdk-ecs = "=1.133.1"`); `main.rs` validates the resolved region via a testable `require_region` helper BEFORE creating AWS clients and exits non-zero with a clear message when none is found; `client.rs` drops its silent `.or_else("us-east-1")` fallback.

Implements QUAL-07 and QUAL-08.
</objective>
