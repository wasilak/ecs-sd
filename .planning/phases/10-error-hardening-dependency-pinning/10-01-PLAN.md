<objective>
Harden the two production HTTP paths that currently panic on response construction and add outbound-connection timeouts to the shared reqwest client.

Purpose: A `.unwrap()` panic in an async handler aborts the Tokio task (not the process), silently dropping the request and leaking a stack trace; a reqwest client with no connect timeout can accumulate TCP black-hole connections that exhaust file descriptors under concurrent scrapes. Both are operational-excellence defects for a service that runs unattended behind an ALB.

Output: `proxy_handler` and `metrics_handler` return HTTP 500 (static message, no internal detail) instead of panicking on a failed `Response::builder().body(...)`; the `reqwest::Client` in `AppState::new` is built with a 5s connect timeout and 10s TCP keepalive.

Implements QUAL-03 and QUAL-04.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| upstream target → proxy_handler | Untrusted upstream HTTP response (status, headers, body) crosses into response construction |
| Prometheus scraper → metrics_handler | Scrape request triggers encode + response construction |
| proxy_handler → upstream target (outbound TCP) | Outbound connection to an ECS task IP that may vanish without FIN/RST |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Denial of Service | `proxy_handler` / `metrics_handler` response construction | mitigate | Replace `.unwrap()` on `Response::builder().body(...)` with `unwrap_or_else` returning HTTP 500 — a construction failure no longer panics/aborts the Tokio task (QUAL-03) |
| T-10-02 | Information Disclosure | `proxy_handler`/`metrics_handler` 500 fallback body | mitigate | Fallback returns a static string only; the `http::Error` value is logged via `warn!`, never written into the HTTP response body (QUAL-03) |
| T-10-03 | Denial of Service | Outbound `reqwest::Client` in `AppState` | mitigate | `connect_timeout(5s)` bounds the TCP handshake; `tcp_keepalive(10s)` (SO_KEEPALIVE) detects black-hole peers so sockets are reclaimed instead of exhausting file descriptors (QUAL-04) |
