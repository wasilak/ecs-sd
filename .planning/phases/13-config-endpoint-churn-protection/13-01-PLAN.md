Add churn protection guard to `replace_cache_and_record_metrics` in src/state/app_state.rs:

1. **Add tracing import** — At the top of the file, add `use tracing::warn;` (currently app_state.rs does not import tracing; see handlers/sd.rs line 13 for the project pattern of importing specific macros).

2. **Insert churn guard** — AFTER the `new_addresses` HashSet is computed (line 207) and BEFORE the `self.replace_cache_and_routing(targets_aws).await;` call (line 209), insert:
   ```
   // CHURN-01: Guard against excessive target drop
   let max_drop_ratio = self.config.max_target_drop_ratio;
   if max_drop_ratio > 0.0 && !old_addresses.is_empty() {
       let drop_count = old_addresses.difference(&new_addresses).count();
       let drop_ratio = drop_count as f64 / old_addresses.len() as f64;
       if drop_ratio > max_drop_ratio {
           warn!(
               drop_count,
               old_count = old_addresses.len(),
               new_count = new_addresses.len(),
               drop_ratio = format_args!("{:.4}", drop_ratio),
               max_drop_ratio = format_args!("{:.4}", max_drop_ratio),
               "target churn exceeds threshold — discarding refresh, keeping stale cache"
           );
           return;
       }
   }
   ```

3. **Add churn guard unit tests** in the existing `#[cfg(test)] mod tests` block at the end of app_state.rs. These tests do NOT need a real AppState (they test the churn logic directly by testing the address diff math). However, since the guard is embedded in `replace_cache_and_record_metrics` which needs AppState, write the tests as follows:

   Add a helper function `churn_guard_should_discard(old_addrs: &HashSet<String>, new_addrs: &HashSet<String>, max_drop_ratio: f64) -> bool` as a `pub(crate)` function that encapsulates just the churn check logic. Then `replace_cache_and_record_metrics` calls this helper. Tests exercise the helper directly:
