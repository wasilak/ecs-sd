<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → /config handler | Unauthenticated client requests config data (auth is out of scope per project design) |
| Config struct → ConfigResponse | Serialization boundary where refresh_token secret must be excluded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Information Disclosure | config_handler leaking refresh_token | mitigate | ConfigResponse struct does NOT have a refresh_token field — defense in depth via separate serialization struct (Pitfall 3 from RESEARCH.md) |
| T-13-05 | Information Disclosure | Config response exposing internal state to unauthenticated callers | accept | Auth is out of scope per REQUIREMENTS.md "Out of Scope" section; network-level controls recommended |
| T-13-06 | Tampering | Config endpoint could be used to fingerprint version for targeted attacks | accept | Version exposure is intentional for debugging; same data available via /health endpoint |
</threat_model>
