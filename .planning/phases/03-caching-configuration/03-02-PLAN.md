## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| periodic background worker → AWS APIs | remote dependency failures/throttling can influence refresh success |
| concurrent HTTP readers → shared cache locks | untrusted request volume competes with refresh writes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Denial of Service | refresh scheduler in `src/main.rs` | mitigate | Apply ±10% jitter per D-01, `MissedTickBehavior::Skip`, and stale-while-revalidate fallback to prevent synchronized API bursts and request blocking. |
| T-03-05 | Tampering | cache freshness signaling | mitigate | Compute `X-Cache-Age` from shared `last_refresh` timestamp updated only on successful atomic refresh; do not mutate on failed refresh. |
| T-03-06 | Repudiation | runtime refresh events | mitigate | Emit explicit INFO/WARN/DEBUG logs (`started`, `complete`, `failed`, `cache hit/miss`) with consistent message strings for operator auditability. |
</threat_model>
