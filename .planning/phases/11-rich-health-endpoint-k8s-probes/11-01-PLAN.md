<objective>
Extend `AppState` with the state needed to describe the service's operational health: process start time and the outcome (success/failure + timestamp) of the most recent discovery refresh attempt. Wire the background refresh loop and the initial discovery block in `main.rs` to record that outcome after every attempt.

Purpose: The current `CacheSnapshot` is only written on a *successful* refresh (the stale-on-failure guarantee from Phase 9). There is therefore no record anywhere of a *failed* refresh attempt. The rich `/health` endpoint (Plan 02) needs both the uptime and a failure signal to compute status and the HTTP 503 condition (HEALTH-02). This plan creates those data sources; it does not read them yet.

Output: `RefreshOutcome` struct, two new `AppState` fields, a re-export, a `unix_now()` helper, and four outcome-write sites in `main.rs`.
</objective>
