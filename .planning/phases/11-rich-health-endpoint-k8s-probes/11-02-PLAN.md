<objective>
Replace the stateless `/health` handler with a rich, structured operational status endpoint, and add two simple status endpoints: `/health/live` (always-200 liveness) and `/health/ready` (cache-gated readiness). All status/HTTP-code logic is extracted into pure functions that are unit-tested inline.

Purpose: Operators and load balancers need three distinct signals. `/health` reports full operational state (uptime, cache, cluster topology, last refresh). `/health/live` reports only that the process is alive — it must never depend on cache or AWS state, so a transient AWS outage cannot cause the instance to be evicted. `/health/ready` gates traffic on whether the cache holds usable discovery data.

Output: three handlers, four `#[derive(Serialize)]` response structs, two pure decision functions, inline unit tests, and three registered routes.
</objective>
